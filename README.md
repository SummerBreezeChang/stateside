# Stateside

Stateside helps an international student compare three U.S. rentals they already found and decide what to verify before applying, paying, or signing.

> Marketplaces help students find places. Guides explain the rules. Stateside helps a student understand whether these three specific places work for their situation—and shows what it does not know.

## The judge flow

1. **Setup:** Begin with a labeled UC Berkeley sample student and three editable rental excerpts.
2. **Compare:** See the same ten questions for every place, with qualification first and every answer labeled Confirmed, Inferred, or Unknown.
3. **Decision plan:** Open a place-specific plan with five review sections, one action lane, three next actions, documents, landlord questions, and a copyable inquiry email.

The landing page names the outcome, explains the mechanism, and shows the workflow in three visual steps before setup. The first three seconds of comparison prioritize qualification, true cost, commitment, then unknowns. Guests can save favorites locally on their device without creating an account.

No account, live inventory, payment, safety score, or personal documents are required. Public listing imagery is used as evidence: photo coverage and reused images are visible, exact routes remain labeled as estimates, and every gallery warns that photos do not replace a live tour.

## Why it is different

Existing products are strong at inventory, student booking, university portals, guarantor coverage, or roommate discovery. Stateside is a source-neutral decision layer after discovery and before transaction. It focuses on four gaps:

- Normalizing costs and dates into the same rows and units.
- Treating unknown facts as visibly as known ones.
- Showing qualification before an application fee is paid.
- Being able to recommend pausing because Stateside has no booking revenue.

See [COMPETITOR_ANALYSIS.md](./COMPETITOR_ANALYSIS.md) for the primary-source market review.

## How GPT-5.6 is central

The model interprets messy listing text, landlord messages, and lease excerpts into a strict JSON schema. It separates confirmed statements, reasonable inferences, and unknowns; explains why cautions matter; identifies qualification routes; surfaces contradictions; and prepares questions and an inquiry email.

The application—not the model—calculates monthly totals, move-in cash, and lease/program date differences. The complete sample experience renders from [`site/fixtures/analysis.json`](./site/fixtures/analysis.json), a real structured response generated by one GPT-5.6 Responses API request. This fixture-first design keeps the judge path reliable without making a paid network request during the demo.

The strongest sample reveal is a Fulton room advertised with “utilities included.” Its lease excerpt separately bills electricity and makes each tenant responsible for unpaid co-tenant balances. Stateside preserves the contradiction instead of silently choosing the more favorable interpretation.

## Safeguards

- Never invent property, landlord, location, or transit facts.
- Never convert an unknown into a reassuring inference.
- Never determine whether a home, person, or neighborhood is safe.
- Never use protected characteristics or demographic proxies.
- Never give legal or immigration conclusions.
- Never guarantee approval or legitimacy.

Stateside does not determine whether a home or neighborhood is safe. It helps students review available evidence, understand uncertainties, and prepare appropriate questions before committing.

## Local setup

Requirements: Node.js 22.13 or newer.

```bash
cd site
npm install
npm run dev
```

The pre-generated sample works without credentials. To regenerate the structured fixture, create `site/.env.local` with `OPENAI_API_KEY` and run `npm run fixture:generate`. Environment files are ignored by Git.

## Testing

```bash
cd site
npm run lint
npm test
```

Automated checks cover the server-rendered one-click setup, structured fixture contract, hidden-clause evidence, fixed comparison-row order, defensive states, deterministic calculation wiring, and disclaimer. The complete interaction has also been tested in a desktop and 390px-wide browser viewport.

## Architecture

- `site/data/analysis-schema.json` — strict model output contract
- `site/data/sample-input.json` — safe three-place Berkeley scenario
- `site/fixtures/analysis.json` — saved real GPT-5.6 structured output
- `site/fixtures/markets.json` — dated Bay Area market observations
- `site/fixtures/media.json` — local gallery paths, coordinates, map links, and reused-image evidence
- `site/public/listings/` — 28 self-hosted public listing images used by the visual evidence layer
- `site/app/layout.tsx`, `robots.ts`, and `sitemap.ts` — public title, search, sharing, and indexing metadata
- `site/lib/analysis.ts` — deterministic calculations and validation
- `site/app/page.tsx` — three-screen fixture-driven experience
- `site/scripts/generate-fixture.mjs` — reproducible one-call fixture generator
